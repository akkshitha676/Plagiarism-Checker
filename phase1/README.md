# Phase 1 – Tokenization and Preprocessing

## Overview

Phase 1 establishes the foundation of the plagiarism detection system by implementing source code tokenization and preprocessing. It converts C++ programs into a structured representation that can be used for similarity analysis in later phases.

---

## Directory Structure

```text
phase1/
├── Makefile
├── main.cpp
├── tokenizer.cpp
├── match_submissions.hpp
└── testcases/
```

---

## Components

| File | Description |
|------|-------------|
| `main.cpp` | Entry point for the Phase 1 implementation |
| `tokenizer.cpp` | Implements lexical tokenization of C++ source code |
| `match_submissions.hpp` | Interface for submission matching |
| `testcases/` | Sample programs used for validating the tokenizer |

---

## Building

Compile the project using the provided Makefile.

```bash
make
```

---

## Running

Execute the binary generated by the Makefile.

```bash
./main
```

---

## Objective

The primary objective of this phase is to preprocess C++ source code into a normalized representation suitable for plagiarism detection. This preprocessing pipeline serves as the basis for the comparison algorithms introduced in subsequent phases.

---

## Progression

The implementation developed in this phase is extended in **Phase 2**, where tokenized source code is compared to identify similarities between program submissions.
